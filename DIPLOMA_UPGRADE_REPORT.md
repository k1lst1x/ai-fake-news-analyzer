# Diploma Upgrade Report

## 1) Общие сведения

- Модельная часть: Transformer + Random Forest meta-model.
- Мультиязычность: EN (основной), RU/KK через локальный перевод в EN.
- Контекстная аналитика: история проверок и агрегаты в SQLite (`data/analytics.db`).

## 2) Требование по датасету (15-20 ГБ)

Реализован полный пайплайн расширения:

1. `scripts/expand_dataset.py` — сбор seed-корпуса из локальных и внешних источников.
2. `scripts/scale_dataset_to_target.py` — масштабирование до целевого объёма с контролируемой аугментацией и сохранением меток.

### Фактический результат (выполнено)

- Файл: `data/expanded/expanded_news_dataset_15gb.csv`
- Размер: **15.236 GB**
- Строк: **12,602,435**
- Класс fake: **4,674,570**
- Класс real: **7,927,865**

Требование «увеличить с 5 ГБ до 15-20 ГБ» закрыто.

## 3) Explainable AI (XAI)

Реализовано в `backend/xai.py` и выдаётся через API/UI:

- Вероятность фейка (в процентах)
- Причины:
  - явные признаки (кликбейт-лексика, паттерн заголовка)
  - неявные признаки (эмоциональная форма, неопределённые формулировки, дефицит источников)
  - контекстные признаки (уверенность модели и согласованность с RF)
- Подсветка подозрительных фрагментов
- Компактные коды сигналов (`compact_codes`) и минимальный hint для агента

Формат соответствует требованию:
- «Вероятность фейка: N%»
- «Причины: ...»
- Подсветка подозрительных частей текста

## 4) Веб-интерфейс

`app.py` полностью переработан:

- Страница 1: Landing
  - описание проекта и целей
  - демонстрация real/fake примеров
  - кнопка «Попробовать»
- Страница 2: Аналитический дашборд
  - общее число проверок
  - fake/real по времени
  - география (если доступно поле `country`)
  - языковое распределение
  - фильтры: период, язык, тип контента
  - экспорт CSV/JSON
- Страница «Проверка новости»
  - анализ текста
  - XAI-объяснение
  - подсветка фрагментов

Дизайн:
- тёплая цветовая схема
- мягкие градиенты и скругления
- адаптивность для desktop/tablet/mobile

## 7) Оптимизация на минимальное потребление токенов

- Добавлены компактные коды причин (`compact_codes`)
- Добавлен короткий агентный шаблон (`agent_hint_min_tokens`)
- Отдельный файл с минимальным форматом ответа: `AGENT_HINT_MIN_TOKENS.md`

Краткая схема:
- FAKE при `fake_prob >= 0.60` или наличии 2+ сильных сигналов
- REAL при `fake_prob < 0.45` и отсутствии сигнала `LOW_SOURCE_EVIDENCE`
